# How To Use Me Subtree

This subtree contains team-facing notes about George's working style. It is intentionally scoped and should not be treated as repo-wide guidance.

## Loading strategy

- Start with `README.md` in this directory.
- Load only the files needed for the current task.
- Do not load every file in this subtree unless the user asks for a full review or rewrite.

## Editing rules

- Do not treat preserving the author's exact wording or voice as a default constraint. Preserve or explicitly challenge the underlying operating intent unless the user asks to change it.
- Prefer minimal edits over broad rewrites.
- Avoid replacing specific heuristics with generic management-language summaries.
- Before adding or accepting a more generic statement, ask for the rationale behind it and the concrete scenarios it is meant to cover.
- Ask clarifying questions when the requested change is ambiguous or would benefit from tighter scope.
- Raise concerns when the user's text is too broad, unfalsifiable, or strips away trade-offs that matter.
- Keep topic boundaries stable unless the user asks for a reorganization.
- If you move content across files, update `README.md` so the routing stays accurate.
